# 记忆系统优化设计方案

## 一、架构概述

### 1. 记忆系统分层
- **角色记忆系统**：每个 NPC 独立的记忆库
- **上帝记忆系统**：用于编写上帝小说的全局记忆库

### 2. 核心设计原则
- **记忆独立性**：角色之间记忆互不污染
- **记忆分级**：1-5级记忆强度
- **时序性**：记忆按时间顺序存储和检索
- **可编辑性**：所有记忆可手动查看和修改
- **智能管理**：自动记忆整理和优先级调整

## 三、API 设计

### 1. 记忆整理 API

#### 角色记忆整理
- **端点**：`/api/memory/character/process`
- **参数**：
  - `characterId`: 角色ID
  - `memories`: 当前记忆列表
  - `newEvents`: 新事件列表
- **返回**：整理后的记忆结构

#### 上帝记忆整理
- **端点**：`/api/memory/god/process`
- **参数**：
  - `globalEvents`: 全局事件
  - `characterInteractions`: 角色互动
  - `timeRange`: 时间范围
- **返回**：整理后的上帝记忆和故事摘要

### 2. 记忆评分 API

- **端点**：`/api/memory/rate`
- **参数**：
  - `content`: 记忆内容
  - `context`: 上下文
  - `characterId`: 角色ID
- **返回**：
  - `score`: 1-5级评分
  - `emotion`: 情感标签
  - `importance`: 重要性评估

## 四、记忆管理机制

### 1. 记忆生命周期
- **1-2级记忆**：默认保存7天，无重复提及自动降级
- **3级记忆**：默认保存30天，定期降级
- **4-5级记忆**：长期保存，除非手动删除

### 2. 记忆提取策略
- **情境匹配**：基于当前环境和对话内容提取相关记忆
- **时间权重**：近期记忆优先级更高
- **情感关联**：与当前情感状态相关的记忆优先
- **重要性排序**：高等级记忆优先

### 3. 记忆更新机制
- **新事件触发**：每次互动后更新记忆
- **定期整理**：每天自动整理记忆库
- **手动调整**：用户可手动修改记忆等级和内容

## 十一、备选方案

### 方案一：本地记忆处理
- **特点**：所有记忆处理在本地完成，无需外部 API
- **优势**：响应速度快，无需网络连接
- **劣势**：处理能力有限，复杂分析效果不佳

### 方案二：混合记忆系统
- **特点**：本地存储 + 云端分析
- **优势**：兼顾响应速度和分析能力
- **劣势**：需要网络连接，实现复杂

### 方案三：完全云端记忆
- **特点**：所有记忆存储和处理在云端
- **优势**：强大的分析能力，跨设备同步
- **劣势**：依赖网络，响应延迟

## 十二、推荐方案

**推荐方案二：混合记忆系统**

- **理由**：
  - 本地存储确保基本功能的响应速度
  - 云端分析提供更智能的记忆管理
  - 平衡了性能和功能需求
  - 符合用户对记忆系统的复杂要求

- **关键实现**：
  - 本地存储核心记忆和近期记忆
  - 云端处理记忆整理和评分
  - 定期同步确保数据一致性
  - 离线模式下使用本地处理能力
